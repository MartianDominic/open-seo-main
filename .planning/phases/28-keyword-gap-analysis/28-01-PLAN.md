# Task 28-01: Competitor Discovery & Domain Intersection API

## Objective

Integrate DataForSEO's Domain Intersection API to discover competitors and identify keyword gaps. This task implements the API wrapper for `/v3/dataforseo_labs/google/domain_intersection/live` endpoint and adds schema support for storing gap analysis results.

## Tasks

### 1. Add schema for keyword gaps and competitors (TEST FIRST)
- [ ] Write test for `KeywordGap` interface validation
- [ ] Add `KeywordGap` interface to `prospect-schema.ts`
- [ ] Add `competitorDomains` array field to analysis schema
- [ ] Add `keywordGaps` JSONB field to `prospectAnalyses` table
- [ ] Generate and apply Drizzle migration

### 2. Implement Domain Intersection API wrapper (TEST FIRST)
- [ ] Write test for `fetchDomainIntersectionRaw()` with mock response
- [ ] Add `domainIntersectionItemSchema` to `dataforseoSchemas.ts`
- [ ] Implement `fetchDomainIntersectionRaw()` in new file `dataforseoKeywordGap.ts`
- [ ] Add type exports for `DomainIntersectionItem`
- [ ] Test validates cost tracking and billing info

### 3. Integrate with DataForSEO client (TEST FIRST)
- [ ] Write test for `client.prospect.domainIntersection()` method
- [ ] Add `domainIntersection()` method to dataforseoClient prospect namespace
- [ ] Verify billing/metering integration works correctly
- [ ] Test validates error handling for invalid domains

### 4. Add opportunity scoring logic (TEST FIRST)
- [ ] Write test for `calculateOpportunityScore()` function
- [ ] Implement `calculateOpportunityScore(gap: KeywordGap): number`
  - Formula: `searchVolume * cpc * (100 - difficulty) / 100`
  - Returns traffic potential weighted by difficulty
- [ ] Sort gaps by opportunity score descending
- [ ] Test edge cases (zero volume, null CPC, etc.)
