# Plan 34-03: Keyword Aggregation Service

## Objective

Create a KeywordAggregationService that merges keywords from multiple sources into a unified list for keyword-page mapping. This service collects keywords from GSC query snapshots, saved keywords, keyword rankings, and prospect analyses (if client converted from prospect).

## Context

@34-CONTEXT.md (if exists)

The keyword-page mapping system needs a unified keyword pool to map to URLs. Keywords come from:
1. **GSC query snapshots** - Last 30 days, min 10 impressions
2. **Saved keywords** - User-saved keywords with metrics
3. **Keyword rankings** - Current ranking positions
4. **Prospect analyses** - If client converted from prospect, include gap/opportunity keywords

## Tasks

### Task 1: Create KeywordAggregationService with types

type: auto

Create the service file with TypeScript types and the main aggregation interface.

**Done criteria:**
- `src/server/services/keyword-aggregation/KeywordAggregationService.ts` exists
- `AggregatedKeyword` type defined with source attribution
- `KeywordSource` union type defined
- Service class skeleton with `aggregateForProject` method signature

### Task 2: Implement GSC query aggregation

type: auto

Fetch keywords from `gsc_query_snapshots` table:
- Filter: last 30 days, impressions >= 10
- Group by query (case-insensitive)
- Calculate avg position, total impressions, total clicks across days
- Track source as "gsc"

**Done criteria:**
- `aggregateFromGSC` private method implemented
- Filters by date range (30 days) and min impressions (10)
- Returns deduplicated keywords with metrics

### Task 3: Implement saved keywords aggregation

type: auto

Fetch from `saved_keywords` joined with `keyword_metrics`:
- Include search volume, CPC, difficulty
- Track source as "saved"

**Done criteria:**
- `aggregateFromSavedKeywords` private method implemented
- Joins with keyword_metrics for metrics
- Returns keywords with tracking info

### Task 4: Implement rankings aggregation

type: auto

Fetch from `keyword_rankings` table:
- Get latest position for each keyword
- Track source as "ranking"

**Done criteria:**
- `aggregateFromRankings` private method implemented
- Gets most recent ranking per keyword
- Returns keywords with current position

### Task 5: Implement prospect analysis aggregation

type: auto

If client was converted from prospect, include keywords from their analysis:
- Gap keywords (competitor keywords we don't rank for)
- Opportunity keywords (AI-generated)
- Track source as "prospect_analysis"

**Done criteria:**
- `aggregateFromProspectAnalysis` private method implemented
- Checks `clients.convertedFromProspectId` to find prospect
- Loads `prospect_analyses.keywordGaps` and `opportunityKeywords`
- Returns keywords with achievability scores

### Task 6: Implement deduplication and merge logic

type: auto

Merge all sources into unified list:
- Case-insensitive keyword matching
- Merge metrics from multiple sources
- Track all sources keyword appears in
- Priority: current position > saved metrics > prospect data > GSC

**Done criteria:**
- `aggregateForProject` orchestrates all sources
- Deduplication by normalized keyword (lowercase, trimmed)
- Multi-source attribution preserved
- Position data prioritized when available
