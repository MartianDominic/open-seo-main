CSV import with papaparse parsing, Zod validation, and preview dialog for bulk prospect creation.

## Implementation Details

### Task 1: CSV Validation Schema and Parser

Created `src/server/lib/csv/` module with:

- **prospectCsvSchema.ts**: Zod schema for CSV row validation
  - Domain normalization (strips protocol, www, path, port, lowercases)
  - CSV injection sanitization (prefixes =, +, -, @ with single quote per OWASP)
  - Email validation (optional field, rejects malformed)
  - Deduplication by normalized domain

- **prospectCsvParser.ts**: Papaparse wrapper with header normalization
  - Supports header variations: company/company_name, email/contact_email, name/contact_name
  - Returns ParseResult with valid rows, invalid rows, totalRows, duplicatesRemoved

- **prospectCsvParser.test.ts**: 10 unit tests covering:
  - Domain validation (required, normalization)
  - Email validation (optional, format check)
  - CSV injection sanitization
  - Deduplication logic
  - Header normalization variations

### Task 2: Server Function

Added to `src/serverFunctions/prospects.ts`:

- **importCsvSchema**: Validates array of prospect rows, max 10,000 (T-30.5-03)
- **importProspectsFromCsv**: Creates prospects in batch
  - Handles CONFLICT errors gracefully (counts as skipped)
  - Tracks created, skipped, and error counts
  - Source defaults to "csv_import" for tracking

### Task 3: CsvImportDialog Component

Created `src/client/components/prospects/CsvImportDialog.tsx`:

- File upload area with drag-drop styling
- Client-side validation mirrors server schema for instant feedback
- Preview table shows first 10 valid rows
- Invalid rows displayed with row numbers and error messages
- Duplicate count shown
- Import button shows count of prospects to create
- Success/error alert after import completes
- Refreshes prospects list on successful import

Updated prospects page (`src/routes/_app/prospects/index.tsx`):
- Added "Import CSV" button in header
- Wired CsvImportDialog with state management
- List refreshes via React Query invalidation after import
