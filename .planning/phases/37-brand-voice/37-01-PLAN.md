Create voice-schema.test.ts first with failing tests validating:
1. Table column existence and types using Drizzle's type inference
2. Enum values exported correctly
3. JSONB type definitions match expected interfaces

Then implement voice-schema.ts with:

**voiceProfiles table:**
- id: text PK (nanoid)
- clientId: text FK -> clients.id (cascade delete)
- mode: text (preservation/application/best_practices) - per D-01
- tone_primary: text
- tone_secondary: text (nullable)
- formality_level: integer (1-10)
- personality_traits: jsonb array of strings
- archetype: text (professional/casual/technical/friendly/authoritative)
- sentence_length_avg: integer
- paragraph_length_avg: integer
- contraction_usage: text (never/sometimes/frequently)
- vocabulary_patterns: jsonb { preferred: string[], avoided: string[] }
- signature_phrases: jsonb array
- forbidden_phrases: jsonb array
- heading_style: text (title_case/sentence_case/all_caps)
- confidence_score: integer (0-100)
- analyzed_at: timestamp (nullable)
- created_at, updated_at: timestamps

**voiceAnalysis table:**
- id: text PK
- profileId: text FK -> voiceProfiles.id (cascade delete)
- url: text (page URL analyzed)
- raw_analysis: jsonb (full AI response)
- extracted_tone: text
- extracted_formality: integer
- sample_sentences: jsonb array
- created_at: timestamp

**contentProtectionRules table:**
- id: text PK
- profileId: text FK -> voiceProfiles.id (cascade delete)
- rule_type: text (page/section/pattern)
- target: text (URL, CSS selector, or regex depending on rule_type)
- expires_at: timestamp (nullable)
- reason: text
- created_by: text (user ID)
- created_at: timestamp

Add indexes on clientId, profileId columns. Export const arrays for enum values.
